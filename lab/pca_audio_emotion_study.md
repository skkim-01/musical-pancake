# PCA와 음성 감정 분석 학습 정리

## 1. PCA (Principal Component Analysis)

### 1.1 핵심 개념

**"분산이 큰 방향 = 정보가 많은 방향"**

- 분산이 크다 = 데이터 포인트들을 서로 구별할 수 있다
- 분산이 0이면 모든 데이터가 동일 → 정보 가치 없음

### 1.2 주성분을 찾는 과정

1. **목표**: 어떤 방향 w로 투영했을 때 분산이 최대가 되는가?
2. **문제**: 벡터 길이를 늘리면 분산이 무한대로 → 방향만 찾고 싶으니 **단위벡터 제약** (||w|| = 1)
3. **제약 있는 최적화** → 라그랑주 승수법 사용
4. **결과**: 공분산 행렬의 **고유값 문제**로 귀결

### 1.3 고유값 문제

$$\mathbf{C}\mathbf{w} = \lambda\mathbf{w}$$

- **C**: 공분산 행렬
- **w**: 고유벡터 = 주성분 **방향**
- **λ**: 고유값 = 그 방향의 **분산**

### 1.4 주성분 선택

| 주성분 | 의미 |
|--------|------|
| PC1 | 가장 큰 고유값의 고유벡터 (분산 최대) |
| PC2 | 두 번째로 큰 고유값의 고유벡터 (PC1과 직교) |
| PC3 | 세 번째로 큰 고유값의 고유벡터 (PC1, PC2와 직교) |
| ... | ... |

**모든 주성분은 서로 직교** → 각각 독립적인 정보를 담음

### 1.5 주성분 개수 결정 (휴리스틱)

1. **누적 설명 분산**: 95% 설명하는 데 필요한 개수
2. **Scree Plot**: 고유값 그래프에서 급격히 완만해지는 지점 (elbow)
3. **Kaiser 기준**: 고유값 > 1인 것만 선택

## 2. 라그랑주 승수법

### 2.1 왜 필요한가?

- **제약 없을 때**: 미분=0인 점이 최대/최소 후보
- **제약 있을 때**: 제약 위의 최적점은 기울기가 0이 아닐 수 있음

### 2.2 핵심 아이디어

제약 위의 최적점에서는:
- **"목표 함수 증가 방향"**과 **"제약 조건 방향"**이 **수직**

수식으로:
$$\nabla f = \lambda \nabla g$$

### 2.3 절차

1. 라그랑지안 구성: L = (최대화할 것) − λ × (제약조건)
2. 모든 변수에 대해 미분 = 0
3. 해 후보들 중 최대값 선택

## 3. 음성 처리 파이프라인

```
원본 음성 (1D: 시간 × 샘플값)
        │
        ▼
[1] 전처리 (노이즈 제거)
        │
        ▼
[2] 프레이밍 (슬라이딩 윈도우, 20-40ms)
        │
        ▼
[3] FFT → 주파수 스펙트럼 (2D: 시간 × 주파수)
        │
        ▼
[4] Mel Filter Bank
        │
        ▼
[5] 로그
        │
        ▼
[6] DCT → MFCC (13개 계수)
        │
        ▼
[7] 통계 요약 → 고정 크기 벡터
        │
        ▼
[8] 분류 모델 (SVM, Random Forest 등)
        │
        ▼
감정 분류 결과
```

## 4. FFT vs STFT

### 4.1 FFT (전체 적용)

- 입력: 전체 음성 신호
- 출력: 하나의 주파수 스펙트럼
- **문제**: "언제" 어떤 소리가 났는지 정보 손실

### 4.2 STFT (Short-Time Fourier Transform)

- 프레이밍 후 각 프레임에 FFT 적용
- 출력: 2D 행렬 (시간 × 주파수)
- **장점**: 시간 정보와 주파수 정보 둘 다 유지

```
[아] [버] [지] [가] ...
 │    │    │    │
FFT  FFT  FFT  FFT
 ↓    ↓    ↓    ↓
스펙  스펙  스펙  스펙
```

## 5. Mel Scale & Mel Filter

### 5.1 Mel Scale

**인간 청각 특성 반영**:
- 저주파: 100Hz vs 200Hz → 확실히 다르게 들림
- 고주파: 8000Hz vs 8100Hz → 거의 똑같이 들림

→ **저주파는 촘촘하게, 고주파는 넓게** 필터 배치

### 5.2 Mel Filter Bank

**삼각형 필터**로 가중합 계산:
- 중심 주파수에 가중치 높음
- 양 끝으로 갈수록 가중치 낮음
- 인접 필터끼리 겹침 → 부드러운 전환

```
필터 배치:
    ╱╲   ╱╲   ╱╲   ╱╲
   ╱  ╲ ╱  ╲ ╱  ╲ ╱  ╲
  ╱    ╳    ╳    ╳    ╲
─────────────────────────→ 주파수
저주파(촘촘)    고주파(넓음)
```

수천 개 주파수 → **26개 정도**로 압축

## 6. MFCC (Mel-Frequency Cepstral Coefficients)

### 6.1 전체 흐름

```
FFT 결과 (수천 개 주파수)
        │
        ▼
Mel Filter Bank (26개)
        │
        ▼
로그 (인간 청각: 소리 크기를 비율로 느낌)
        │
        ▼
DCT (12-13개)
```

### 6.2 로그를 취하는 이유

인간 귀는 **소리 크기를 로그 스케일**로 느낀다:
- 10 → 100: "많이 커졌다"
- 1000 → 1100: "거의 비슷"

### 6.3 DCT (Discrete Cosine Transform)

**"Mel Filter 출력 곡선을 코사인 패턴들의 조합으로 분해"**

$$c_k = \sum_{n=0}^{N-1} x_n \cos\left[\frac{\pi}{N}\left(n + \frac{1}{2}\right)k\right]$$

- k가 작으면: 느린 코사인 → 곡선의 **큰 형태** 포착
- k가 크면: 빠른 코사인 → **세밀한 떨림** 포착 (노이즈)

**앞쪽 13개만 사용** (뒤쪽은 노이즈)

### 6.4 MFCC 최종 결과

```
프레임 1개 → [c1, c2, c3, ..., c13]

전체 음성:
프레임1: [c1, c2, ..., c13]
프레임2: [c1, c2, ..., c13]
...
프레임N: [c1, c2, ..., c13]

→ N × 13 행렬
```

---

## 7. 통계 요약

### 7.1 왜 필요한가?

- 사람마다 말 속도가 다름 → 프레임 개수 다름
- 모델 입력은 **고정 크기**가 필요

### 7.2 방법

각 MFCC 계수(c1~c13)에 대해:

| 통계 | 의미 |
|------|------|
| 평균 | 전반적인 값 |
| 분산 | 얼마나 변했나 |
| 최대/최소 | 극단값 |

```
c1의 N개 값 → 평균, 분산, 최대, 최소 → 4개 숫자
c2의 N개 값 → 평균, 분산, 최대, 최소 → 4개 숫자
...
c13의 N개 값 → 평균, 분산, 최대, 최소 → 4개 숫자

총: 13 × 4 = 52개 숫자 (고정 크기)
```

## 8. 분류 모델

### 8.1 SVM (Support Vector Machine)

**"두 그룹을 가장 잘 나누는 경계면을 찾는다"**

- 경계면에 가장 가까운 점들 = Support Vector
- 다중 클래스: One-vs-All 또는 One-vs-One 방식

```
    ●           ○
  ●   ●      ○   ○
    ●    ╱    ○
  ●     ╱   ○   ○
       ╱
      ╱  ← 결정 경계
```

### 8.2 Random Forest

**"결정 트리 여러 개 만들어서 다수결"**

**결정 트리**: 특징값으로 예/아니오 질문 반복
```
        c1_평균 > 5?
       ╱          ╲
     Yes           No
     ╱               ╲
c3_분산 > 2?      c7_최대 > 10?
  ╱    ╲            ╱    ╲
기쁨   슬픔       분노    중립
```

**분기 조건 선택 기준**: 지니 불순도 (Gini Impurity)
- 나눈 후 각 그룹이 얼마나 깔끔해지는가?
- 불순도가 가장 많이 줄어드는 조건 선택

**Random Forest**:
- 트리 여러 개 생성 (데이터/특징 랜덤 샘플링)
- 다수결로 최종 결정
- 과적합 방지

### 8.3 모델 선택

정해진 공식 없음. **실험으로 비교**:
1. 여러 모델 시도
2. 하이퍼파라미터 튜닝
3. 교차 검증
4. 성능 비교 (정확도, F1-score 등)

## 9. 핵심 용어 정리

| 용어 | 의미 |
|------|------|
| 샘플링 레이트 | 초당 샘플 수 (44.1kHz = 초당 44100번) |
| 프레이밍 | 신호를 짧은 윈도우로 자르기 |
| FFT | 시간 → 주파수 변환 |
| STFT | 짧게 잘라서 FFT (시간 정보 유지) |
| Mel Scale | 인간 청각 기준 주파수 스케일 |
| MFCC | 음성의 특징을 13개 숫자로 요약 |
| 공분산 행렬 | 변수들 간의 상관관계 |
| 고유벡터 | 행렬 곱해도 방향 안 바뀌는 벡터 |
| 고유값 | 고유벡터의 스케일 변화량 |

## 10. 실습 코드 예시

```python
import librosa
import numpy as np

# 음성 로드 (librosa 내장 샘플)
y, sr = librosa.load(librosa.ex('trumpet'))

# MFCC 추출
mfcc = librosa.feature.mfcc(y=y, sr=sr, n_mfcc=13)
print(f"MFCC shape: {mfcc.shape}")  # (13, 프레임수)

# 통계 요약
mfcc_mean = np.mean(mfcc, axis=1)
mfcc_std = np.std(mfcc, axis=1)
mfcc_max = np.max(mfcc, axis=1)
mfcc_min = np.min(mfcc, axis=1)

# 고정 크기 특징 벡터
features = np.concatenate([mfcc_mean, mfcc_std, mfcc_max, mfcc_min])
print(f"Feature vector shape: {features.shape}")  # (52,)
```
